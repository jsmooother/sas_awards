Reward search needs a browser session cookie. To get it:
1. Log in at virginatlantic.com (Flying Club).
2. Open DevTools (F12) → Network → Fetch/XHR. Run a reward flight search (e.g. London → New York).
3. Click the graphql request (SearchOffers) → Headers tab.
4. Best: Under Request Headers, find Cookie → right‑click its value → Copy value. Paste below. (One long line – often 2000+ characters; that’s normal.)
5. Or: Under Response Headers you’ll see 4 Set-Cookie lines. Paste all 4 here; we’ll extract and use them.

If the Cookie value is truncated in DevTools, right‑click the Cookie row → Copy value to get the full string.

Which of the 4 Set-Cookie headers does the API need?

Virgin’s response sends 4 Set-Cookie headers. We don’t know the minimal set; the API may need one or more of:

  • com.virginholidays.att – tracking
  • com.virginatlantic.edge.id – edge/session id
  • bm_s – Akamai Bot Manager (long value)
  • bm_sv – Akamai Bot Manager (shorter)

So either copy the single Request Cookie header (recommended), or paste all 4 Response Set-Cookie lines; we use everything you paste.
